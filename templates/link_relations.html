FacePass

Outlines the custom link relations used within the FacePass.

FacePass:register

Registers a new identity with their facial data and permissions.

POST /identities/register

FacePass:permissions

Get all the permissions related to an Identity.

GET /identities/{user_id}/permissions

FacePass:access-requests

Get access requests that belong to the identity.

POST /identities/{user_id}/requests

edit

Updates the details (name, permissions, facial data) of an existing user.

PUT /identities/{user_id}/update

delete

Deletes user's profile and all associated data from the system.

DELETE /identities/{user_id}/delete

FacePass:access-logs

Retrieves access logs for a specified user.

GET /identities/{user_id}/access-logs

FacePass:get-by-name

Retrieve users by their name, along with their associated access permissions.

GET /identities/{user_name}/profile

FacePass:get-by-id

Retrieves the profile of the identity by the ID.

GET /identities/{user_id}/profile

FacePass:request-access

Handle an access request using facial recognition to grant or deny access.

POST /identities/access-request

FacePass:log

Get the access log of the access request.

GET /access-log/{log_id}

FacePass:access-by

User details of the access request.

GET /identities/{user_id}/profile

FacePass:by-access

Get the access request corresponding to access request.

GET /access-request/{access_request_id}

FacePass:log-by

The user profile of the log.

GET /identities/{user_id}/profile

FacePass:permission_by

Profile that owns the permission.

GET /identities/{user_id}/profile
